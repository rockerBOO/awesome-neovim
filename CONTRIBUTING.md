## Submitting Issues

Please follow these instructions in a sequential order:

1. Ensure the issue has not already been reported by searching on GitHub under [Issues][issues].
2. If you're unable to find an open issue addressing the problem, open a new one.
3. Be sure to include a concise title and description.
4. Include as much relevant information as possible.

---

## Submitting Pull Requests

When submitting a Pull Request make sure to cover all of the following points:

1. Ensure your changes follow the [awesome list guidelines][guidelines].
2. Ensure the plugin you're requesting meets our [acceptance criteria][criteria].
3. Ensure your Pull Request title follows the [regexp rule][regexp_rule] (``` Add|Update|Remove `username/repo` ```).
